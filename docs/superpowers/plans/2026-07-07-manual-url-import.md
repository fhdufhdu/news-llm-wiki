# Manual URL Import Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace RSS collection with a web-only multi-URL article import flow that stores raw HTML, shows total and URL-level progress, and triggers Codex wiki updates.

**Architecture:** Spring Boot remains layered: controllers call services, services coordinate repositories and infrastructure, repositories own SQLite access. RSS runtime code is removed from the active source tree; existing historical Flyway migrations stay intact. Manual import jobs create article rows and raw HTML rows, then existing Codex wiki generation updates wiki tables.

**Tech Stack:** Spring Boot 4, Thymeleaf, SQLite, Flyway, Java HTTP fetch infrastructure, Codex CLI, Python helper generated by `CodexWikiJobBuilder`.

## Global Constraints

- Controllers must compose services only, not repositories.
- Package order remains `application / service / repository / infrastructure / dto`.
- Server mechanically fetches URLs and stores raw HTML; LLM only updates wiki data.
- Section navigation uses fixed navigation rows; Codex-created wiki sections are shown in the right-side section list.
- Source-listing features are removed from the active runtime.
- YAGNI, KISS, DRY.
- RSS-related runtime code and UI copy must be removed, not merely hidden.
- Existing applied Flyway migrations are not deleted.

void updatesItemStatusAndProgressCounts() {

    long itemId = repository.findItems(jobId).getFirst().id();

    repository.markFetching(itemId);
    repository.markFetched(itemId, "https://example.com/a", 10L);

    assertThat(job.wikiDoneCount()).isEqualTo(1);
    assertThat(job.failedCount()).isZero();
    assertThat(item.status()).isEqualTo("DONE");

}

    id INTEGER PRIMARY KEY AUTOINCREMENT,
    status TEXT NOT NULL DEFAULT 'PENDING',
    total_count INTEGER NOT NULL DEFAULT 0,
    fetched_count INTEGER NOT NULL DEFAULT 0,

CREATE TABLE IF NOT EXISTS article_import_items (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    job_id INTEGER NOT NULL,
    input_url TEXT NOT NULL,
    canonical_url TEXT,
    article_id INTEGER,
    status TEXT NOT NULL DEFAULT 'PENDING',
    error_message TEXT,
    attempt_count INTEGER NOT NULL DEFAULT 0,
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at TEXT NOT NULL DEFAULT (datetime('now')),
    FOREIGN KEY(job_id) REFERENCES article_import_jobs(id),
    FOREIGN KEY(article_id) REFERENCES articles(id)
);

CREATE INDEX IF NOT EXISTS idx_article_import_items_job_status ON article_import_items(job_id, status);
CREATE INDEX IF NOT EXISTS idx_article_import_items_article ON article_import_items(article_id);
